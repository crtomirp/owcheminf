# v0.2.1 Release Runbook

This is the concrete command-by-command runbook for the next patch release after `v0.2.0`.

## Assumptions

- repository: `https://github.com/crtomirp/owcheminf`
- main branch: `main`
- conda environment: `owcheminf`
- package version to release: `0.2.1`

## 1. Sync and inspect the branch

```bash
cd /Users/crtomir/Desktop/cinf
git checkout main
git pull --ff-only
git status
```

## 2. Update release metadata

Make sure these files are updated before tagging:

- `pyproject.toml` -> `version = "0.2.1"`
- `CHANGELOG.md` -> add `## [0.2.1] - YYYY-MM-DD`
- optional docs or release notes if needed

Quick checks:

```bash
rg -n '^version = ' pyproject.toml
rg -n '^## \\[0\\.2\\.1\\]' CHANGELOG.md
git diff -- pyproject.toml CHANGELOG.md
```

## 3. Run the release validation suite

```bash
conda activate owcheminf
python -m pip install -e .[dev]
python -m compileall -q src tests
python -m pytest -q
python -m build
python -m unittest discover -s tests -p 'test_*smoke.py' -v
python -m pytest tests/test_widget_smoke_tester.py tests/test_widget_import_smoke.py -q
PYTHONPATH=src python -m chem_inf_widgets.chemcore.tools.clean_repo_cli
git status
```

## 4. Commit release preparation

```bash
git add pyproject.toml CHANGELOG.md README.md docs src .github
git commit -m "Prepare v0.2.1 release"
```

## 5. Create and push the tag

Manual route:

```bash
git tag -a v0.2.1 -m "Release v0.2.1"
git push origin main
git push origin v0.2.1
```

Or use the helper script:

```bash
./scripts/release_tag.sh 0.2.1
```

## 6. Publish the GitHub release

After pushing the tag:

1. open `https://github.com/crtomirp/owcheminf/releases/new?tag=v0.2.1`
2. set a clear release title
3. compress the changelog into short highlights
4. publish the release

The `Release Assets` workflow should attach `sdist` and `wheel` files automatically.

## 7. Publish to TestPyPI first

Open Actions in GitHub:

1. run `Publish PyPI`
2. choose `repository = testpypi`
3. set `ref = v0.2.1`

Then verify the uploaded package page and install flow.

## 8. Publish to PyPI

After TestPyPI validation:

1. run `Publish PyPI`
2. choose `repository = pypi`
3. set `ref = v0.2.1`

## 9. Post-release checks

```bash
git show --no-patch --decorate v0.2.1
git ls-remote --tags origin | rg 'v0.2.1'
```

Then verify:

- GitHub release exists
- release assets are attached
- PyPI page is correct
- install instructions still work
