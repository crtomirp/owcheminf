# v0.2.1 Plan

This is the working checklist for the next patch release after `v0.2.0`.

## Goal

Stabilize the new GitHub-facing release process and continue tightening QSAR, widget workflow, and packaging reliability.

## Candidate Scope

- QSAR widget stability and UX polish
- target and prediction metadata consistency
- release and packaging automation hardening
- documentation cleanup for GitHub and PyPI readiness

## Checklist

### Release hygiene

- [ ] review `CHANGELOG.md`
- [ ] confirm `pyproject.toml` version bump to `0.2.1`
- [ ] review repository URLs and badges
- [ ] confirm `SECURITY.md`, `CONTRIBUTING.md`, and release docs are current

### Packaging

- [ ] run `python -m build`
- [ ] run smoke packaging tests
- [ ] confirm no `src/*.egg-info`, `__pycache__`, or `.pyc` artifacts are tracked
- [ ] confirm GitHub release assets workflow still matches actual package layout

### Widget and workflow checks

- [ ] run widget import smoke tests
- [ ] run `Widget Smoke Tester`
- [ ] verify core workflow smoke still includes `Mol Editor`
- [ ] verify QSAR prediction flow with `SMILES` and `Molecules` inputs

### QSAR/QSPR checks

- [ ] verify `QSAR Regression` manual vs auto mode behavior
- [ ] verify `QSAR/QSPR Model Hub` target auto-detection
- [ ] verify `QSAR Prediction Packager` recipe-aware prediction path
- [ ] verify descriptor and fingerprint recipes remain backward-compatible

### Optional dependency checks

- [ ] verify graceful degradation without `optuna`
- [ ] verify graceful degradation without `mordred`
- [ ] verify docs still match optional dependency behavior

### Release execution

- [ ] create annotated tag `v0.2.1`
- [ ] push `main` and the tag
- [ ] publish GitHub release
- [ ] optionally publish to TestPyPI first
- [ ] publish to PyPI when validated

## Nice-to-have If Time Allows
