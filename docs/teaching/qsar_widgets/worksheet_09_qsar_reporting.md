# Worksheet 09: QSAR Reporting and Reproducibility

**Estimated time:** 60–90 min  
**Level:** intermediate  
**Main widgets:** MLR Model Selection, QSAR Regression, Data Table

## Context

A QSAR result is only useful if another person can understand how it was produced. Reproducibility requires reporting data, descriptors, splitting, model settings, and validation.

## Intended learning outcomes

Students will be able to:

1. write a minimal QSAR methods section,
2. report descriptor/fingerprint settings,
3. report splitting and validation strategy,
4. communicate model limitations.

## Orange workflow

Example:

```text
File → Mol Descriptors 2 → Scaffold Splitter → MLR Model Selection
```

## Student tasks

Prepare a one-page QSAR report containing:

1. dataset description,
2. target variable,
3. descriptor or fingerprint type,
4. train/test strategy,
5. model type,
6. performance metrics,
7. chemical interpretation,
8. limitations.

## Guiding questions

- Could someone repeat your workflow from your report?
- Did you report the SMILES source and target column?
- Did you avoid claiming causality from correlation?
- Did you describe the applicability domain?

## Expected output

A short written report or slide with workflow and results.
