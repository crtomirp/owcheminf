# Worksheet 07 — QSAR Report Generator

## Learning goals

Students learn how to turn QSAR outputs into a reproducible report that documents dataset quality, model metrics, validation, applicability domain, explanations and limitations.

## Orange workflow

```text
QSAR Model Hub → QSAR Validation Dashboard → QSAR Report Generator
Applicability Domain Workbench → QSAR Report Generator
Model Explanation → QSAR Report Generator
```

## Tasks

1. Train a model with QSAR Model Hub.
2. Send predictions to QSAR Validation Dashboard.
3. Send metrics, predictions and summaries to QSAR Report Generator.
4. Inspect the Markdown report.
5. Identify which sections are complete and which are missing.

## Guiding questions

- Which information is required for a reproducible QSAR report?
- Why should applicability-domain results be reported together with metrics?
- Why is a limitation statement necessary?

## Evidence of learning

Students submit the generated report and a short paragraph explaining whether the model is ready for external use.
