# Security Policy

## Reporting a Vulnerability

If you believe you found a security issue, please do not open a public GitHub issue first.

Instead, report it privately to:

- `crtomir.podlipnik@fkkt.uni-lj.si`

Please include:

- a short description of the issue
- steps to reproduce
- affected files, widgets, or workflows
- any proof-of-concept material that helps verify the problem

## What To Expect

- You should receive an acknowledgement as soon as practical.
- The issue will be reviewed and triaged.
- If confirmed, a fix and coordinated disclosure plan will be prepared.

## Scope Notes

This project is mainly a scientific desktop add-on for Orange, so the most likely security-relevant areas are:

- bundled web/editor assets
- file import/export behavior
- remote data retrieval integrations
- dependency vulnerabilities

Operational environment issues outside this repository are out of scope unless they are directly triggered by the package itself.
